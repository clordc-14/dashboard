# 国药西南新药引进网开发设计文档

## 1. 项目概述

本项目建设一个用于展示新药引进相关信息的报表展示网页，网站标题为 **“国药西南新药引进网”**。页面支持用户上传 Excel 文件，系统自动读取 Excel 中的新闻信息、品种明细、引进协助事项等关键内容，并生成面向业务人员和管理层浏览的可视化展示页面。

页面不直接展示上传 Excel 原文件的超链接，避免暴露本地文件路径或原始附件地址。对于表格类板块，可提供“查看完整表格”入口，该入口跳转到系统内自动生成的表格详情页或锚点区域，而不是直接打开原始 Excel 文件。

## 2. 建设目标

1. 支持上传 `.xlsx` / `.xls` 格式的 Excel 文件。
2. 自动识别并读取 Excel 中指定业务板块的数据。
3. 生成两个动态新闻板块：
   - 本周重点信息
   - 上周上市创新药回顾
4. 生成五个表格展示板块，展示 Excel 中提取出的关键表格数据。
5. 新闻板块支持跳转到新闻源地址。
6. 表格板块支持跳转到系统内生成的完整表格详情，并可定位到对应数据信息区域。
7. 页面风格应适合医药企业内部经营看板，强调清晰、稳重、可读、便于快速浏览。

## 3. 推荐技术栈

### 3.1 前端基础

- HTML5：页面结构。
- CSS3：页面布局、响应式样式、打印样式。
- JavaScript：文件上传、Excel 解析、数据清洗、页面渲染、交互逻辑。

### 3.2 推荐依赖

- SheetJS `xlsx`：在浏览器端读取和解析 Excel 文件。
- ECharts：如后续需要增加统计图、进度图、占比图，可用于图表展示。
- Vite：本地开发与构建工具，适合轻量前端项目。
- dayjs：日期格式化和新闻时间排序。

### 3.3 可选后端

如果仅用于本地或内网静态展示，可采用纯前端方案，不需要后端。

如果需要保存历史文件、多人访问、权限控制或定时更新，建议增加后端：

- Node.js + Express：提供文件上传、数据解析、接口输出。
- SQLite / MySQL：存储解析后的新闻、表格、上传记录。
- Nginx：部署静态页面和反向代理接口。

## 4. 页面结构设计

### 4.1 首页

首页为核心展示页，推荐结构如下：

1. 顶部标题栏
   - 网站标题：国药西南新药引进网
   - 上传 Excel 按钮
   - 数据更新时间

2. 新闻展示区
   - 本周重点信息
   - 上周上市创新药回顾

3. 表格板块区
   - 已建档品种明细
   - 需领导协助引进（未合作厂牌）
   - 其他从 Excel 中读取的表格板块 1
   - 其他从 Excel 中读取的表格板块 2
   - 其他从 Excel 中读取的表格板块 3

4. 表格详情区
   - 用户点击某个表格板块后，进入详情页或滚动定位到完整表格。
   - 支持搜索、筛选、分页、列排序。

### 4.2 新闻板块

每个新闻板块展示内容包括：

- 新闻标题
- 摘要或重点内容
- 发布时间
- 来源名称
- 新闻源链接

交互要求：

- 点击新闻标题或“查看来源”按钮，打开新闻源网页。
- 新闻链接必须来自 Excel 中配置的源地址字段，或由后台人工维护。
- 如果某条新闻没有源链接，页面应显示为普通文本，不提供跳转。

### 4.3 表格板块

每个表格板块展示内容包括：

- 板块标题
- 关键指标摘要，例如品种数量、待协助数量、重点品种数量等。
- 表格前若干行预览，建议展示 5 到 10 行。
- 查看完整表格按钮。

表格板块不应直接暴露上传 Excel 的文件链接。点击“查看完整表格”后，应跳转到系统内生成的表格详情，例如：

```text
/table.html?section=archived-products
#section-need-leader-support
```

## 5. Excel 数据读取设计

### 5.1 文件上传

用户通过页面上传 Excel 文件：

```html
<input type="file" accept=".xlsx,.xls" />
```

前端使用 `FileReader` 读取文件内容，再通过 SheetJS 解析：

```javascript
const reader = new FileReader();
reader.onload = function (event) {
  const data = new Uint8Array(event.target.result);
  const workbook = XLSX.read(data, { type: "array" });
};
reader.readAsArrayBuffer(file);
```

### 5.2 数据识别规则

建议使用“配置化板块映射”方式，而不是把 sheet 名、列名写死在页面逻辑中。

示例配置：

```javascript
const dashboardConfig = {
  newsSections: [
    {
      key: "weeklyFocus",
      title: "本周重点信息",
      matchTitle: "本周重点信息",
      type: "news"
    },
    {
      key: "lastWeekInnovativeDrugs",
      title: "上周上市创新药回顾",
      matchTitle: "上周上市创新药回顾",
      type: "news"
    }
  ],
  tableSections: [
    {
      key: "archivedProducts",
      title: "已建档品种明细",
      matchTitle: "已建档品种明细"
    },
    {
      key: "needLeaderSupport",
      title: "需领导协助引进（未合作厂牌）",
      matchTitle: "需领导协助引进（未合作厂牌）"
    }
  ]
};
```

系统读取 Excel 后，根据配置中的 `matchTitle` 查找对应 sheet 名、标题行或区域标题，再提取该区域下方的表格数据。

### 5.3 新闻数据字段建议

新闻板块建议从 Excel 中读取以下字段：

| 字段 | 含义 | 是否必填 |
|---|---|---|
| title | 新闻标题 | 是 |
| summary | 新闻摘要或重点内容 | 否 |
| sourceName | 新闻来源 | 否 |
| sourceUrl | 新闻源链接 | 否 |
| publishDate | 发布时间 | 否 |
| category | 所属板块 | 是 |

### 5.4 表格数据字段建议

表格字段以 Excel 实际列为准，常见字段包括：

| 字段 | 含义 |
|---|---|
| productName | 品种名称 |
| companyName | 生产企业 / 厂牌 |
| status | 当前状态 |
| cooperationStatus | 合作状态 |
| owner | 负责人 |
| progress | 引进进展 |
| remark | 备注 |

由于 Excel 表格可能存在合并单元格、空行、说明行，解析时需要做数据清洗：

1. 跳过完全空白行。
2. 自动识别标题行。
3. 合并单元格应补齐到对应数据行。
4. 表头字段应统一转换为系统内部字段名。
5. 链接字段需要校验是否为合法 URL。

## 6. 页面交互设计

### 6.1 上传 Excel

上传成功后页面显示：

- 文件解析成功提示。
- 数据更新时间。
- 已识别的新闻板块数量。
- 已识别的表格板块数量。

如果解析失败，应显示明确错误原因：

- 文件格式不支持。
- 未识别到指定板块。
- 表头缺失。
- 新闻源链接格式异常。

### 6.2 新闻跳转

新闻标题和来源按钮都可跳转到新闻源。跳转建议使用新窗口打开：

```html
<a href="https://example.com/news" target="_blank" rel="noopener noreferrer">
  查看来源
</a>
```

### 6.3 表格详情跳转

表格预览板块中的“查看完整表格”按钮跳转到系统内详情区域：

```javascript
function openTableSection(sectionKey) {
  window.location.href = `/table.html?section=${sectionKey}`;
}
```

如果是单页应用，也可以使用页面内锚点定位：

```javascript
document
  .querySelector(`#section-${sectionKey}`)
  .scrollIntoView({ behavior: "smooth" });
```

## 7. 信息安全与文件处理要求

1. 页面不得展示上传 Excel 文件的本地路径。
2. 页面不得直接暴露原始 Excel 文件下载链接。
3. 表格详情应基于解析后的数据生成，而不是直接打开原文件。
4. 新闻源链接只允许跳转到合法的 `http` 或 `https` 地址。
5. 如部署到内网服务器，上传文件应设置大小限制，例如不超过 20MB。
6. 如涉及敏感经营数据，应增加访问权限控制。

## 9. 核心模块说明

### 9.1 excelParser.js

职责：

- 读取 Excel 文件。
- 遍历 workbook 中的 sheet。
- 提取单元格数据、合并单元格信息和链接信息。
- 输出统一的中间数据结构。

### 9.2 sectionMatcher.js

职责：

- 根据配置查找“本周重点信息”“上周上市创新药回顾”等板块。
- 根据标题或 sheet 名识别五个表格板块。
- 返回每个板块所在 sheet、起始行、结束行和原始数据范围。

### 9.3 normalizer.js

职责：

- 清洗空行。
- 统一字段名。
- 处理日期格式。
- 校验新闻链接。
- 补齐合并单元格带来的空值。

### 9.4 newsRenderer.js

职责：

- 渲染两个新闻板块。
- 支持新闻源跳转。
- 支持空状态展示。

### 9.5 tableRenderer.js

职责：

- 渲染五个表格预览板块。
- 渲染完整表格详情。
- 支持搜索、筛选、分页、排序。

## 10. 验收标准

1. 页面标题显示为“国药西南新药引进网”。
2. 页面可正常上传并解析 Excel 文件。
3. 页面不显示 Excel 原始文件超链接或本地路径。
4. 页面能生成两个新闻板块：
   - 本周重点信息
   - 上周上市创新药回顾
5. 新闻板块中的新闻可跳转到对应新闻源。
6. 页面能生成五个表格板块。
7. 表格板块能展示关键数据预览。
8. 点击表格板块可查看完整表格或定位到完整数据区域。
9. Excel 中没有对应数据时，页面显示友好的空状态。
10. 页面在常见桌面分辨率下展示正常，表格不应出现内容严重溢出。

1. 增加历史上传记录，支持按日期回看往期报表。
2. 增加关键词搜索，例如按品种名称、厂牌、负责人搜索。
